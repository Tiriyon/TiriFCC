Here I have learned about variables in CSS. A variable is declared: --var-name: property; which then can be called as a propety on other arguments like so: background: var(--var-name);

A fallback variable is an argument that makes the browser fall back to if the variable is missing in the CSS file. It is used like so: background: var(--black-skin, black);

Compatibility and fallback

If I ran into browser compatibility issues, an easy fix is to probide a fallback option. The browser parses the CSS ignoring any properties that it doesn't recognize or support. For example, if the CSS variable to assign a background color on a site is ignored on IE (which does not work with variables) - it will not load the variable and ignore that property. If it can't find any other value set for that property, it will revert to the default value, which is typically not ideal.

Henc, I can provide a fallback by adding a property above the one that does not render on the browser on the same CSS selector.

:root{ --red-color: red;}

.red-box { background: red; This is the fallback.

background: var(--red-color);}

Root is a pseudo-class selector that matches the root element of the document, usually the html element. By creating my variables in :root, they will be available globally and can be accessed from any other selector in the style sheet.

Note that when you create a variable, it is available for you to use inside the selector in which you create it. It also is available in any of that selector's descendants. (inheritance)
